# ADR-0006: i18n Strategy

## Status

draft

## Date

2026-05-15

## Context

The game ships German-first with English maintained from day one. The exact
key structure and pluralization needs depend on Wave 2 UI research.

## Options Considered

- i18next / react-i18next, German default, English maintained (intended).
- Custom lightweight key map.
- Deferred i18n (German hardcoded for MVP).

## Decision

(Intended, pending Wave 2) Use i18next/react-i18next with German as default
language and English resources maintained from day one.

## Rationale

i18next is the mature React standard with pluralization and namespace support;
maintaining English from day one avoids a costly retrofit later.

## Consequences

Positive:

- Stable translation keys; English-ready without a rewrite.

Negative:

- Feature copy must live in locale resources; UI code must use stable keys.
  Key taxonomy unstable until Wave 2 UI research closes.
